## 步骤一：创建 Azure Container Registry (ACR)

ACR 是 Azure 的 Docker 镜像仓库，用于存储我们构建的容器镜像。

```powershell
az acr create \
  --resource-group xkf_kafka \
  --name megatronacr0404 \
  --sku Basic \
  --location eastasia
```

**结果**：创建成功，登录服务器为 `megatronacr0404.azurecr.io`

**为什么需要 ACR**：K8s Pod 需要从镜像仓库拉取 Docker 镜像。ACR 与 AKS 在同一 Azure 区域，拉取速度快且可以通过 RBAC 授权免密拉取。

---

## 步骤二：创建 AKS 集群

AKS 是 Azure 托管的 Kubernetes 服务，自动管理控制面。

```powershell
az aks create \
  --resource-group xkf_kafka \
  --name megatron-aks \
  --location eastasia \
  --node-count 1 \
  --node-vm-size Standard_DS2_v2 \
  --generate-ssh-keys \
  --no-wait
```

**参数说明**：
- `--node-count 1`：1个工作节点（学习用，节省成本）
- `--node-vm-size Standard_DS2_v2`：2核 CPU、7GB 内存（无 GPU，用 CPU 模拟）
- `--generate-ssh-keys`：自动生成 SSH 密钥
- `--no-wait`：异步创建，不阻塞命令行（约需 5-8 分钟）

**生产环境差异**：
- 真实 AI 训练会使用 GPU 节点（如 `Standard_NC24ads_A100_v4`）
- 多节点 + 节点池（system pool + GPU pool）
- 启用 InfiniBand 加速网络

## 步骤三：编写 Megatron Hello World 程序

创建练习文件夹和 Python 程序：

```
megatron-practice/
├── megatron_hello_world.py   # 主程序
├── Dockerfile                # Docker 打包文件
└── k8s-job.yaml              # K8s 部署清单
```

### 程序内容

`megatron_hello_world.py` 演示了 Megatron-LM 的 7 个核心 AI Infra 概念：

1. **运行环境检测** — GPU 型号、显存、SM 数量、CUDA 版本、K8s 环境变量
2. **分布式初始化** — 模拟 `initialize_megatron()`，NCCL/Gloo 后端选择
3. **张量并行 (TP)** — Column Parallel 和 Row Parallel 的矩阵切分与通信
4. **流水线并行 (PP)** — 1F1B Schedule 的 warmup/steady/cooldown 可视化
5. **分布式优化器** — ZeRO-like 内存节省的量化估算（18B/param → 6+12/DP）
6. **通信模式** — all-reduce / all-gather / reduce-scatter / P2P / all-to-all
7. **分布式 Checkpoint** — ShardedTensor 元数据、异步写入、Resharding

## 步骤四：编写 Dockerfile

```dockerfile
FROM pytorch/pytorch:2.11.0-cuda12.6-cudnn9-runtime

WORKDIR /app
COPY megatron_hello_world.py .
CMD ["python", "megatron_hello_world.py"]
```

**关键决策**：
- 使用 `runtime` 而非 `devel`（更小的镜像，不需要编译工具）
- 基于 PyTorch 官方镜像，已包含 CUDA/cuDNN/PyTorch

---

## 步骤五：构建并推送 Docker 镜像

由于本地 Docker daemon 未运行，使用 **ACR Build**（Azure 云端构建）：

```powershell
cd megatron-practice

az acr build \
  --registry megatronacr0404 \
  --image megatron-hello-world:v1 \
  --file Dockerfile .
```

**ACR Build 的好处**：
- 不需要本地安装/运行 Docker
- 构建在 Azure 云端完成，带宽更快
- 构建完成后自动推送到 ACR

**构建过程**（约 6 分钟）：
1. 上传源码 tar 包 (5 KiB)
2. 拉取基础镜像 pytorch/pytorch (~3 GB)
3. 复制应用代码
4. 推送镜像到 ACR

---

## 步骤六：授权 AKS 拉取 ACR 镜像

AKS 需要权限从 ACR 拉取镜像：

```powershell
# 等待 AKS 创建完成
az aks wait --resource-group xkf_kafka --name megatron-aks --updated

# 附加 ACR 到 AKS（自动配置 RBAC 角色）
az aks update \
  --resource-group xkf_kafka \
  --name megatron-aks \
  --attach-acr megatronacr0404
```

**原理**：这个命令会给 AKS 的 managed identity 授予 ACR 的 `AcrPull` 角色，使 K8s 节点可以免密拉取镜像。

## 步骤八：编写 K8s Job 并部署

```yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: megatron-hello-world
spec:
  backoffLimit: 2
  template:
    spec:
      containers:
      - name: megatron
        image: megatronacr0404.azurecr.io/megatron-hello-world:v1
        resources:
          requests:
            cpu: "250m"
            memory: "512Mi"
          limits:
            cpu: "500m"
            memory: "1Gi"
        env:
        - name: WORLD_SIZE
          value: "1"
        - name: RANK
          value: "0"
      restartPolicy: Never
```

**K8s 概念说明**：
- **Job**（而非 Deployment）：一次性运行任务，适合训练作业
- **backoffLimit: 2**：失败最多重试 2 次
- **resources.requests**：K8s 调度器根据此值选择节点
- **resources.limits**：容器最大可用资源
- **env**：注入分布式训练所需的环境变量

**部署**：
```powershell
kubectl apply -f k8s-job.yaml
```

**注意事项**：
- 首次遇到 `Insufficient cpu` 错误 — Standard_DS2_v2 只有 2 核，系统组件占用约 1.5 核
- 解决方案：将 requests 降低到 250m CPU / 512Mi 内存

## 📝 遇到的问题与解决

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| Docker 镜像标签不存在 | `pytorch:2.3.1-cuda12.1-cudnn9-runtime` 已下架 | 更换为 `2.11.0-cuda12.6-cudnn9-runtime` |
| 本地 Docker 不可用 | Docker Desktop 未运行 | 改用 ACR Build 云端构建 |
| ACR attach Etag 冲突 | AKS 刚创建完有并发操作 | 等待操作完成后自动生效 |
| Pod Insufficient CPU | 节点 2 核被系统组件占用 | 降低 requests 到 250m |
